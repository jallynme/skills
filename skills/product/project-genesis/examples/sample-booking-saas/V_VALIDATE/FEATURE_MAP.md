# Feature Map — Sample-Booking (filled example)

> Lean start in action: the founder gave **domain + 3 big-picture features**; the swarm researched and
> proposed 23; the founder scope-locked **9** for v1. Owned by CPO + Competitor Analyst, Red-Team on scope.

## 1. Founder input (the big picture)
- **Domain:** appointment-booking SaaS for solo service providers.
- **Big-picture features I want:** (1) a shareable booking link, (2) take a deposit, (3) auto reminders.
- **Hard constraints / non-goals:** no team/staff calendars in v1; no POS/retail.

## 3. The wedge (explicit)
- **We win because:** **deposit-to-confirm with auto-refund**, done beautifully on **mobile** (push +
  one-tap) — everyone else treats payments as a bolt-on and has weak mobile. Everything else is parity.

## 4. Chosen scope → walking skeleton
- **v1 / walking skeleton (Must):** booking link · deposit-to-confirm+refund · reminders · calendar sync ·
  availability rules · cancel policy · client list · mobile push · branded page (9 features).
- **Next (Phase 2):** group bookings · staff calendars · waitlist.
- **Per surface:** Web = booking page + settings + dashboard · Mobile = booking + push + client list · Admin = none (solo).

## 5. Sign-off
- Red-Team scope challenge: "client list is a CRM rabbit hole." Answered: scoped to name+history+rebook only.
- **Founder decision (scope locked for v1):** ✅ approved — 2026-06-28.
